  每个 PLAN.md 文件包含：

  Plan 执行后生成 SUMMARY.md 记录实际完成情况。

⏺ Task 执行触发流程

  从 Plan 生成到 Task 执行的完整链路：

  ┌─────────────────────────────────────────────────────────────────────┐
  │  /gsd:execute-phase [phase]                                        │
  └─────────────────────────────┬───────────────────────────────────────┘
                                │
                                ▼
  ┌─────────────────────────────────────────────────────────────────────┐
  │  Step 1: discover_and_group_plans                                   │
  │  - gsd-tools phase-plan-index → 获取所有 Plan 及其 Wave 依赖          │
  │  - 跳过已有 SUMMARY.md 的 Plan (已完成)                              │
  │  - 按 Wave 分组: Wave1(无依赖) → Wave2(等Wave1) → Wave3(等Wave2)   │
  └─────────────────────────────┬───────────────────────────────────────┘
                                │
                                ▼
  ┌─────────────────────────────────────────────────────────────────────┐
  │  Step 2: execute_waves                                              │
  │                                                                     │
  │  For each Wave:                                                     │
  │    1. 描述要构建的内容 (从 Plan objective 提取)                       │
  │    2. Spawn gsd-executor agent per Plan:                           │
  │       Task(                                                        │
  │         subagent_type="gsd-executor",                              │
  │         model="{executor_model}",                                  │
  │         isolation="worktree",                                       │
  │         prompt="<files_to_read>                                     │
  │           - {phase_dir}/{plan_file}  ← PLAN.md                      │
  │           - .claude/.gsdt-planning/PROJECT.md                                   │
  │           - .claude/.gsdt-planning/STATE.md                                     │
  │           - ./CLAUDE.md                                             │
  │         "                                                           │
  │       )                                                             │
  │    3. 等待 Agent 完成 (或 spot-check 验证)                           │
  │    4. Post-wave: 运行 pre-commit hooks (--no-verify 跳过)          │
  │    5. 验证 SUMMARY.md 存在 + git commits                          │
  │    6. 继续下一个 Wave                                               │
  └─────────────────────────────┬───────────────────────────────────────┘
                                │
                                ▼
  ┌─────────────────────────────────────────────────────────────────────┐
  │  gsd-executor Agent 内部流程:                                        │
  │                                                                     │
  │  1. load_plan → 解析 PLAN.md (frontmatter, tasks, criteria)         │
  │  2. record_start_time                                              │
  │  3. determine_execution_pattern → 检测是否有 checkpoint             │
  │  4. execute_tasks:                                                │
  │     ┌─────────────────────────────────────────────────────────────┐│
  │     │ For each task:                                             ││
  │     │   - type="auto": 执行 → TDD if tdd="true"                  ││
  │     │             → commit (feat|fix|test|refactor|chore)       ││
  │     │             → 跟踪 deviation rules (Rule 1-4)              ││
  │     │   - type="checkpoint:*": STOP → 返回 checkpoint message    ││
  │     │   - 遇到 auth gate → STOP → human-action checkpoint        ││
  │     └─────────────────────────────────────────────────────────────┘│
  │  5. summary_creation → {phase}-{plan}-SUMMARY.md                  │
  │  6. state_updates → advance-plan, update-progress, record-metric  │
  │  7. final_commit → docs({phase}-{plan}): complete plan           │
  │  8. completion_format → PLAN COMPLETE 返回给 orchestrator          │
  └─────────────────────────────────────────────────────────────────────┘

  核心触发机制

  ┌───────────────────┬─────────────────────────────────────────────────────┐
  │       阶段        │                      触发方式                       │
  ├───────────────────┼─────────────────────────────────────────────────────┤
  │ Orchestrator →    │ Task(subagent_type="gsd-executor", ...) — 每个 Plan │
  │ Executor          │  一个 Agent                                         │
  ├───────────────────┼─────────────────────────────────────────────────────┤
  │ Executor 内 Task  │ 顺序遍历 PLAN.md 中的 tasks 数组                    │
  │ 执行              │                                                     │
  ├───────────────────┼─────────────────────────────────────────────────────┤
  │ 原子性保证        │ 每个 task 完成后立即 git commit                     │
  ├───────────────────┼─────────────────────────────────────────────────────┤
  │ 并行执行          │ 同 Wave 内多个 Executor 并行 (PARALLEL=true)        │
  ├───────────────────┼─────────────────────────────────────────────────────┤
  │ 依赖等待          │ Wave N 完成后才启动 Wave N+1                        │
  ├───────────────────┼─────────────────────────────────────────────────────┤
  │ Checkpoint 暂停   │ type="checkpoint:*" → 返回结构化消息 → 等待用户响应 │
  │                   │  → 新 Agent 继续                                    │
  └───────────────────┴─────────────────────────────────────────────────────┘

  Task 类型处理

  type="auto":      # 自动执行直到完成
  type="checkpoint:human-verify":  # 暂停 → 用户验证 UI/功能
  type="checkpoint:decision":      # 暂停 → 用户做决策
  type="checkpoint:human-action":  # 暂停 → 用户执行不可自动化的操作 
  (如邮箱验证)
